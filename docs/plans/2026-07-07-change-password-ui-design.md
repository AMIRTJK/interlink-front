# Дизайн-документ: Добавление функционала смены пароля в профиле

## Проблема / Контекст
Необходимо переименовать кнопку «Редактировать профиль» в «Настройки» и добавить в модальное окно настроек интерфейс для смены пароля пользователя («Сменить пароль»). Интеграция с API бэкенда будет выполнена позже, на данном этапе требуется реализовать только UI.

## Описание изменений

### 1. Переименование кнопки
* В файле `src/features/Profile/ui/tabs/ProfileInfoTab.tsx` меняем текст кнопки с `Редактировать профиль` на `Настройки`.

### 2. Новый компонент смены пароля (`ChangePassword.tsx`)
* Создаем новый компонент `src/features/Profile/ui/ChangePassword.tsx`.
* Добавляем строку с иконкой `KeyRound` и текстом «Пароль безопасности», а также кнопкой «Изменить».
* Кнопка «Изменить» открывает модальное окно `Modal` шириной 360px с формой `Form`.
* Поля формы:
  * Текущий пароль (`Input.Password`, валидация на заполненность).
  * Новый пароль (`Input.Password`, валидация на заполненность и минимальную длину 6 символов).
  * Подтверждение нового пароля (`Input.Password`, валидация на совпадение с новым паролем).
* При успешной валидации формы вызывается имитация успешного сохранения с вызовом `toast.success("Пароль успешно изменен")` и закрытием модального окна.
* В соответствии с системным манифестом разработки (`RULE[GEMINI.md]`) код не будет содержать комментариев.

### 3. Интеграция в модальное окно настроек (`ProfileSettingsModal.tsx`)
* Рендерим новый компонент `<ChangePassword />` внутри `<ProfileSettingsModal>` под `<MfaSecurity />`.
