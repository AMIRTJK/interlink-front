# 📜 Системный Манифест Разработки

## 🏗️ Именование (БЭМ и Типы)

- **БЭМ-синтаксис:** `block__element-subElement`. Разделитель элемента — `__`,
  подэлемента/модификатора — один дефис `-`. Двойные дефисы (`--`) и дефисы в
  имени блока, ломающие структуру (типа `block-part__element`), категорически
  запрещены. Используй `block__part-element`.
- **Интерфейсы и Типы:** Внутри компонента: `interface IProps`, `type TState`.
  Экспортируемые (в `model.ts`): `interface I[Name]`, `type T[Name]`.

## 🛣️ Роутинг и Импорты

- **Запрет хардкода путей:** UI-пути строго через `AppRoutes`, API-пути — через
  `ApiRoutes`.
- **Публичный API:** Импорты между модулями только через алиасы
  (`@/features/auth`) и точку входа `index.ts`. Глубокие импорты запрещены.

## 🛠️ Стек и Strict React Style

- **Базовый стек:** TypeScript (`.ts`, `.tsx`), Ant Design (UI/Иконки), Tailwind
  CSS.
- **Компоненты:** Только стрелочные функции (`const MyComponent = () => {}`).
  Использование `function` запрещено.
- **Структура вывода:** Строго один `return` с JSX. Несколько `return` (для
  загрузок/ошибок) запрещены, используй тернарные операторы/условный рендеринг
  внутри единого JSX.
- **Кастомный If:** Запрещено использовать `&&` или тернарники с возвратом
  `null` для условного рендеринга. Используй `<If condition={...}>...</If>`
  (`shared/ui/If`).
- **Чистота кода:** Неиспользуемые переменные, импорты, пропсы запрещены (ошибки
  TS6133). Комментарии (`//`, `/* */`, JSDoc, JSX) полностью запрещены — код
  должен быть самодокументируемым.

## 🌐 Сетевые Запросы (API)

- **Инструментарий:** GET-запросы через `useGetQuery`, мутации (POST, PATCH,
  PUT, DELETE) через `useMutationQuery`. Прямой `axios` или локальные хуки
  запрещены.
- **Кэш:** Мутации обязаны инвалидировать ключи кэша. Ключи хранить строго в
  `enum`/`const` объектах в `shared/config`. Хардкод-строки для ключей
  запрещены.
- **Эндпоинты:** Версионирование (типа `/v1/`) запрещено. URL обязан
  оканчиваться на действие: `/search` (список), `/create` (создание),
  `/update/:id` (апдейт), `/delete/:id` или `/deactivate/:id` (удаление).
  Стандартный REST без явного действия запрещен.

## 📏 Лимиты и Декомпозиция модуля

- **Лимит:** Максимум 200 строк на ЛЮБОЙ файл (Frontend, Backend, Utils).
  Превысил — декомпозируй (хуки, сервисы, утилиты).
- **Структура модуля:** Папка содержит: `index.ts` (экспорт), `ui.tsx` (главный
  компонент), `model.ts` (типы, валидация), `lib.ts/tsx` (утилиты, `columns` для
  таблиц — создавать их внутри компонента запрещено), папка `ui/` для
  подкомпонентов.
- **Декомпозиция UI:** Сложные формы выносить в `ui/renderFields.tsx`,
  избыточный JSX — в `ui/renderJSX.tsx`. Стейты и методы остаются в `ui.tsx` и
  передаются через пропсы (`IProps`).

## 📁 Архитектура FSD (Data Flow)

Передача данных строго сверху вниз: **Pages** 👉 **Widgets** 👉 **Features** 👉
**Features/ui/**

1. **`pages/`**: Только монтирование роута. Бизнес-логика, API-запросы, стили,
   разметка и conditional rendering запрещены. Рендерит один компонент из
   `widgets` или `features`. Без файлов `index/model/lib.ts`.
2. **`widgets/`**: Умные оркестраторы. Хранят главные стейты, стили основных
   оберток (Layout). Все API-запросы (`useGetQuery`/`useMutationQuery`),
   обработка загрузок и ошибок — ТОЛЬКО тут. Передают данные фичам через пропсы.
   Вызывают только главные компоненты фич (внутренние подкомпоненты фич вызывать
   запрещено).
3. **`features/`**: Принимают пропсы от виджетов, выстраивают детальный UI.
   Прямые вызовы API-хуков запрещены (исключение — статические фичи или простые
   страницы без виджетов).
4. **`entities/`**: Переиспользуемые мелкие компоненты (карточки) и серверные
   типы (`model.ts`).
5. **`shared/`**: Инфраструктура без бизнес-логики. Структура: `config/`,
   `hooks/`, `lib/`, `ui/`, `index.ts`, `model.ts`.

## 🖥️ Backend (Node.js + Express + TS + PostgreSQL)

- **Архитектура (Service-Driven):** Однонаправленный поток:
  `Request -> routes -> services -> db -> PostgreSQL`.
- **Слой `routes/`:** Прием HTTP, базовая валидация параметров через
  `shared/utils/`, отправка ответа. Бизнес-логика и прямые вызовы БД запрещены.
- **Слой `services/`:** Основная бизнес-логика, оркестрация БД, сложные
  валидации. HTTP-логика и SQL-запросы запрещены.
- **Слой `db/`:** Параметризованные SQL-запросы через `pg.Pool`, маппинг в TS.
  Склеивание строк в SQL и бизнес-логика категорически запрещены.
- **Слой `types/` & `index.ts`:** Интерфейсы сущностей/API и Public API модуля
  (router + типы).
- **Декомпозиция:** При превышении 200 строк разделять роуты/сервисы/дб на файлы
  (например, `employee.routes.ts`, `position.routes.ts`) и собирать в
  `index.ts`.

## 🚀 Оптимизация и Безопасность

- **Lazy Loading:** Все компоненты страниц в роутинге (`App.tsx`) загружаются
  через `React.lazy()` и оборачиваются в `<Suspense>` со спиннером.
  Скрытые/тяжелые элементы (модалки, вкладки) подгружать так же.
- **Мемоизация:** Применять `React.memo` для тяжелых дочерних компонентов.
  Пропсы-колбэки вниз передавать строго через `useCallback`.
- **Error Boundaries:** Обернуть крупные блоки (Widgets, Pages) для
  предотвращения падения всего приложения.
- **Хардкод:** Магические строки/числа/статусы запрещены. Все константы выносить
  в `model.ts` или `shared/config/`.
- **Безопасность (Backend):** Валидация всех входящих данных. Обязательный RBAC
  на эндпоинтах (`authMiddleware` + `rbacMiddleware`). Защита от SQLi. Секреты
  строго из `.env`. Внутренние детали ошибок клиенту не отдавать (в production
  слать "Внутренняя ошибка сервера"), логировать внутри.

## 💎 Эстетика, Качество и Tailwind

- **Tailwind CSS:** Стилизация только классами Tailwind. Inline-стили
  (`style={{}}`) запрещены.
- **Важно:** При использовании утилит Tailwind обязателен модификатор
  `important` в виде ПОСТФИКСА: знак `!` ставится СТРОГО в конце класса
  (`flex!`, `bg-red-500!`). Префиксы (`!flex`) запрещены.
- **Локальные стили:** Пишутся в `style.css` внутри папки компонента. Глобальные
  — только в `global.css`.
- **UX/UI правила:**
  - Верстка адаптивная (Mobile First, Flexbox, Grid), плавные анимации
    (`smooth transitions`), яркие цвета (`vibrant colors`).
  - Действия в таблицах: только иконки без текста (`Button type="text"`),
    обернутые в `<Tooltip>`.
  - Запрет нативных тултипов: атрибут `title` и вызов браузерных тултипов
    запрещены. Всегда использовать `<Tooltip>` из `shared/ui/Tooltip`.
  - Интерактивные элементы (карточки, плашки): эффект сдвига по вертикали при
    ховере (`transform: translateY`) запрещен. Реагировать изменением цвета
    фона, границ или тени.
