3. Стек проекта: React, Ant Design (antd) и Tailwind CSS v4.
4. Всегда пиши комментарии к коду на русском языке (короткие, человеческие).
5. Не изменяй логику в MainLayout.tsx без подтверждения.
6. Архитектура: Feature-Sliced Design (FSD). Соблюдай строгое разделение по слоям (app, pages, widgets, features, entities, shared).
7. Импорты: Всегда используй абсолютные пути через алиасы: `@app`, `@pages`, `@widgets`, `@features`, `@entities`, `@shared`.
8. API: Для запросов используй TanStack Query (React Query) и кастомный инстанс `_axios` из `@shared/api`.
9. Именование: Компоненты и папки называй в PascalCase. Используй `index.ts` в папках для чистого экспорта.
10. Иконки: В приоритете `lucide-react`, также допустимы `@ant-design/icons`.
11. Роутинг: Используй константы из `AppRoutes` в `@shared/config/AppRoutes.ts`.
12. Создание модуля (слайса): Папка в PascalCase -> внутри `ui`, `model`, `api` -> `index.ts` для экспорта только нужного.
13. Добавление API: Добавь путь в `api-routes.ts` -> создай Query-хук в `api/` модуля -> используй в компоненте.
14. Перед завершением: Проверь типы и запусти линтер, если были правки в конфигах или API.
15. Переиспользование UI: Перед созданием нового компонента проверь `@shared/ui`. В приоритете: `UniversalTable`, `ActionToolbar`, `FiltersContainer`, `Button`.
16. Стили: Для лейаутов, отступов и быстрых правок используй Tailwind CSS. Для сложных UI-элементов (формы, модалки, таблицы) — Ant Design.
17. Уведомления: Для фидбека пользователю используй `toast` из `react-toastify`.
18. Типизация: Избегай `any`. Всегда типизируй API-ответы и пропсы компонентов.
19. Анимации: Для плавных переходов и микро-взаимодействий используй `framer-motion`.
20. Формы: Для любых форм всегда используй `Form` и `Form.Item` из Ant Design.
21. Конфигурация: Никогда не используй `import.meta.env` напрямую. Всегда бери переменные через `getEnvVar` из `@shared/config`.
22. Декомпозиция: Один модуль/файл — максимум 200 строк. Дели на: главный компонент (контейнер), `renderJSX` (функции/константы для верстки) и вспомогательные компоненты.
23. Архитектурный поток: Данные (API) и пропсы передаются сверху вниз: `page` -> `widgets` -> `features` (функционал). Главная логика блока живет в `widgets`.
24. Именование типов: Используй префиксы: `T` для типов (`TTasks`, `TType`) и `I` для интерфейсов (`ITasks`, `IProps`).
25. Расположение типов: Серверные модели храни в `model/` (или `entities`), локальные типы компонента (пропсы/стейт) — в самом файле компонента (`IProps`, `TType`).
26. Даты: Для работы с датами используй `dayjs`. Избегай нативного `new Date()` для форматирования.
